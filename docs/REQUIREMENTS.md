# 🤖 AI 虚拟伴侣（AI Companion）项目需求文档

## 一、项目概述

本项目旨在开发一款 **高沉浸感、可配置人设、支持记忆与多模态交互的 AI 虚拟伴侣 Web 应用**。  
该应用主要面向特定用户提供长期、专属、拟人化的陪伴聊天体验。

项目核心目标：

1. 提供高颜值、沉浸式聊天体验
2. 支持 AI 人设、行为、语音、视觉形象的后台配置
3. 支持上下文记忆，增强陪伴感与连续性
4. 支持手机和电脑访问，适配多端浏览器
5. 后续可扩展语音、图片生成、主动消息推送等能力

---

## 二、项目定位

这是一个 **专属 AI 陪伴网站**，不是普通聊天机器人页面。  
核心方向是：

- 有“人设感”
- 有“记忆感”
- 有“形象感”
- 有“情绪陪伴感”
- 有“长期对话连续性”

用户进入网站后，看到的不只是一个输入框，而是一个拥有固定身份、语气、外貌设定、欢迎语、聊天风格的 AI 虚拟伴侣。

## 三、技术栈选型

### 1. 前端

- Vue 3
- TypeScript
- Vite
- CSS / SCSS
- 响应式布局（PC + 手机 H5）
- 可选：Pinia、Vue Router、Axios

### 2. 后端

- Java 17
- Spring Boot 3
- RESTful API
- SSE（用于流式输出）

### 3. 数据库

- MySQL 8

### 4. AI 能力接入

- 文本大模型 API（DeepSeek / Kimi / Gemini / OpenAI Compatible）
- TTS 语音合成 API
- ASR 语音转文字 API
- 图像生成 API（Leonardo.ai / SeaArt.ai / 其他）

---

## 四、目标平台与访问方式

### 1. 访问平台

本项目为 **Web 网站**，支持以下终端访问：

- PC 浏览器访问
- 手机浏览器访问
- 平板浏览器访问

### 2. 手机访问要求

网站必须支持手机端正常访问和使用，具体要求如下：

1. 页面采用响应式布局
2. 聊天页面在手机端自动适配宽度与高度
3. 输入框、发送按钮、语音按钮在手机端易于点击
4. 后台管理页在手机端也可以访问，但以“可用”为主，PC 端体验优先
5. 手机端消息区域滚动、自动滚底、打字机效果正常
6. 手机端背景图、毛玻璃效果、头像、气泡布局显示正常
7. 兼容主流移动端浏览器：
   - iPhone Safari
   - Android Chrome
   - 微信内置浏览器（尽量兼容）

### 3. 部署方式

项目部署后，用户可通过域名直接访问。  
后续可部署在：

- 云服务器
- Docker 容器
- Nginx 反向代理

## 五、核心业务目标

### 1. 聊天陪伴

用户可以与 AI 进行自然对话，获得具有连续性的陪伴体验。

### 2. 人设塑造

管理员可以在后台配置 AI 的：

- 名字
- 昵称
- 性格
- 语气
- 背景故事
- 关系设定
- 禁忌规则
- 欢迎语
- 开场白

### 3. 视觉形象塑造

管理员可以在后台配置 AI 的视觉展示内容，包括：

- 头像
- 立绘
- 封面图
- 聊天背景图
- 风格标签
- 外观设定
- 生图提示词
- 多状态情绪图

### 4. 多轮记忆

系统可以记录最近聊天内容，并支持后续长期记忆扩展。

### 5. 多端体验

网站必须兼顾桌面端与手机端用户体验。

## 六、核心功能模块（Phase 1）

### 模块 1：沉浸式聊天页面

#### 功能目标

提供一个高颜值、沉浸式、适合长期聊天陪伴的主界面。

#### 页面组成

1. 顶部 AI 信息区域
2. 中间聊天消息区域
3. 底部输入区域

#### 视觉要求

- 全屏背景图
- 主面板毛玻璃效果（Glassmorphism）
- 半透明背景
- 背景模糊
- 柔和阴影
- 高级感、柔和感、陪伴感

#### 聊天气泡要求

- 类似微信 / QQ 风格
- 用户消息靠右
- AI 消息靠左
- 显示头像
- 支持文本消息
- 预留图片和语音消息展示能力

#### 交互要求

- 支持发送消息
- 支持回车发送（PC）
- 支持点击按钮发送（PC / 手机）
- 支持消息自动滚动到底部
- 支持流式输出打字机效果
- 支持加载中状态
- 支持异常提示

#### 手机端要求

- 输入框固定底部
- 不遮挡聊天区
- 消息内容自动换行
- 气泡宽度适合手机屏幕
- 头像、按钮、文字触控区域合理

---

### 模块 2：隐藏式后台管理页

#### 功能目标

通过隐藏方式进入后台配置页面，用于配置 AI 的各种信息。

#### 进入方式

在聊天输入框中输入隐藏指令，例如：

`/sudo-admin-777`

前端检测到该内容后：

1. 不发送该消息
2. 清空输入框
3. 跳转到后台管理页

#### 后台管理内容

后台用于配置以下内容：

##### A. AI 基础信息

- displayName（显示名称）
- nickname（昵称）
- genderStyle（性别风格）
- ageSetting（年龄设定）
- occupationSetting（职业设定）
- relationshipSetting（关系设定）

##### B. AI 性格与人设

- personalityTags（性格标签）
- speakingStyle（说话风格）
- emotionalStyle（情绪风格）
- tabooRules（禁忌规则）
- characterPrompt（角色设定）
- systemPrompt（系统提示词）

##### C. 模型配置

- modelProvider（模型服务商）
- modelName（模型名称）
- apiKey
- temperature
- topP
- maxTokens
- maxContextCount

##### D. 语音配置

- ttsProvider
- voiceId
- ttsSpeed
- ttsPitch
- enableVoiceChat

##### E. 欢迎与开场配置

- welcomeMessage
- openingScript

##### F. AI 视觉形象配置

- avatarUrl（头像）
- portraitUrl（立绘）
- coverImageUrl（封面图）
- backgroundImageUrl（聊天背景图）
- styleTags（视觉风格标签）
- appearancePrompt（外观设定）
- imagePrompt（生图提示词）
- negativePrompt（负面提示词）
- emotionImages（多状态情绪图）

##### G. 系统全局配置

- 后台隐藏口令
- 默认 AI Profile
- 默认主题色
- 是否开启语音
- 是否开启图片生成
- 是否开启长期记忆
- 是否开启主动推送

---

### 模块 3：AI 角色配置（Profile）

#### 功能目标

支持一套或多套 AI 角色配置，便于切换不同人设。

#### 功能要求

1. 支持新增 AI 角色
2. 支持编辑 AI 角色
3. 支持删除 AI 角色
4. 支持设置默认角色
5. 支持切换当前启用角色
6. 每个角色独立保存：
   - 人设
   - 模型参数
   - 语音参数
   - 视觉形象参数

---

### 模块 4：聊天会话系统

#### 功能目标

支持多会话聊天管理。

#### 功能要求

1. 用户进入页面时创建或加载默认会话
2. 支持会话列表
3. 支持查看单个会话历史
4. 支持修改会话标题
5. 支持删除会话
6. 会话绑定某个 AI Profile

---

### 模块 5：聊天消息系统

#### 功能目标

管理用户与 AI 的消息记录。

#### 消息类型

- text
- image
- audio

#### 角色类型

- user
- assistant
- system

#### 功能要求

1. 保存用户消息
2. 保存 AI 消息
3. 查询历史消息
4. 支持消息扩展字段 extraJson
5. 支持后续图片消息、语音消息扩展

---

### 模块 6：流式输出（SSE）

#### 功能目标

实现 AI 消息逐字输出，提升真实感。

#### 功能要求

1. 前端发送消息后，后端建立 SSE 流
2. 模型输出过程中实时向前端推送文本片段
3. 前端显示打字机效果
4. 结束后保存完整 AI 回复消息
5. 异常时中断并提示

---

### 模块 7：上下文记忆系统

#### 1. 短期记忆

系统每次发送请求时，自动携带最近 N 条聊天记录。

#### 2. 长期记忆（预留）

后续支持将用户的重要信息存入记忆库，例如：

- 用户喜好
- 身份信息
- 常用称呼
- 重要经历
- 特别偏好

#### 记忆展示

可在前端侧边栏或后台中展示记忆内容。

---

## 七、进阶功能模块（Phase 2）

### 模块 1：语音聊天

#### 输入能力

- 用户点击麦克风录音
- 音频上传后端
- 后端调用 ASR 转文字

#### 输出能力

- AI 回复完成后调用 TTS 合成语音
- 前端接收音频并播放

#### 目标效果

让用户可以像“打电话/语音聊天”一样与 AI 互动。

---

### 模块 2：动态视觉反馈 / AI 生图

#### 功能目标

在特定聊天语境下，自动生成符合氛围的视觉内容。

#### 示例

- 用户提到下雨、海边、咖啡馆、晚安等场景
- 后端识别关键词后静默触发生图
- 返回：
  - 角色状态图
  - 场景图
  - AI 漫剧风格卡片

#### 应用位置

- 作为特殊消息插入聊天记录
- 作为背景图切换依据
- 作为角色状态展示

---

### 模块 3：主动消息推送

#### 功能目标

用户长时间未互动时，AI 主动发起问候。

#### 功能方式

- 定时任务
- 超时检测
- 节日 / 时间段触发

#### 示例

- “今天很晚了，还没休息吗？”
- “你今天是不是有点忙，我都没等到你。”

## 八、非功能需求

### 1. 响应式适配

必须兼容：

- 桌面端
- 手机端
- 平板端

### 2. 性能要求

- 首屏加载合理
- 聊天滚动顺畅
- 手机端不卡顿
- 大图资源合理压缩

### 3. 可维护性

- 前后端分层清晰
- 配置项结构清晰
- 代码便于后续扩展模型、语音、生图能力

### 4. 安全性

- 后台页面不直接暴露入口
- API Key 尽量后端存储
- 管理配置接口做权限保护
- 输入内容基础过滤与校验

### 5. 可扩展性

后续支持：

- 多用户
- 多角色
- 多模型切换
- 会员体系
- 消息收藏
- 情绪面板
- 角色皮肤切换

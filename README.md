# 🤖 虚拟伴侣 (AI Companion) 项目需求文档

## 一、项目概述
本项目旨在开发一款高沉浸感、带人设记忆的 AI 伴侣 Web 应用，提供给特定用户（朋友）作为专属聊天对象。项目注重交互体验（高颜值 UI、语音交互、动态视觉）以及高度可配置的后台管理。

## 二、技术栈选型（已确认）
- **前端：** Vue 3（Composition API）+ TypeScript + CSS（重点实现 Glassmorphism 毛玻璃特效）
- **后端：** Java + Spring Boot 3
- **数据库：** MySQL（存储会话记忆与系统配置）
- **AI 接口：** 文本大模型（待定，如 DeepSeek / Kimi / Gemini 等）+ 语音 TTS API + 图像生成 API（Leonardo.ai / SeaArt.ai）

## 三、核心功能模块（Phase 1）

### 1. 沉浸式前端聊天界面（User Interface）
- **视觉风格：** 全屏沉浸式背景，聊天主面板采用毛玻璃效果（Glassmorphism），包含半透明背景、背景模糊（`backdrop-filter`）及柔和阴影。
- **交互细节：**
  - 仿原生微信 / QQ 的聊天气泡布局。
  - 支持大模型流式输出（SSE）的打字机效果。
  - 平滑的自动滚动到底部机制。

### 2. 隐藏式后台管理（Secret Admin Dashboard）
- **触发机制：** 在前端聊天输入框中输入特定隐藏指令（如 `/sudo-admin-777`），前端拦截该消息并清空输入框，无缝路由跳转至配置页。
- **配置项：**
  - **大模型切换：** 可视化切换不同的 LLM 服务商 API Key。
  - **参数微调：** 调节 Temperature（温度值）控制回答的随机性与创造力。
  - **灵魂注入（System Prompt）：** 核心文本域，用于定义女友的人设背景。支持预设剧情向设定（如破镜重圆、傲娇强设、轻松日常等）。
  - **音色选择：** 配置第三方 TTS 的 Voice ID。

### 3. 多模态交互：语音聊天（Voice Chat）
- **输入（ASR）：** 前端调用麦克风权限录音，通过后端转化为文字，发送给大模型。
- **输出（TTS）：** 拦截大模型生成的文本，调用高品质语音合成 API，前端接收音频流并自动播放，实现拟真语音交流。

### 4. 上下文记忆系统（Contextual Memory）
- **短程记忆：** MySQL 记录 `chat_sessions` 与 `chat_messages`，每次对话携带最近 N 条历史记录。
- **长程记忆金库（Memory Vault）：**（预留接口）大模型在后台自动提取用户关键喜好与信息，存入 Vault 数据表，并在前端侧边栏以手账形式呈现。

## 四、进阶功能模块（Phase 2）

### 1. 动态视觉反馈（AI Manga Drama 风格）
- **场景触发：** 识别聊天中的环境或动作关键词。
- **静默生图：** 后端静默调用 Leonardo.ai 或 SeaArt.ai，以 AI 漫剧光影风格生成符合语境的角色状态图或场景卡片，并作为消息发送给用户。

### 2. 主动消息推送（Proactive Messaging）
- 建立定时任务（Cron Job）或超时检测。若用户长时间未互动，后端主动触发大模型生成问候语，打破“仅被动回复”的 AI 刻板印象。
